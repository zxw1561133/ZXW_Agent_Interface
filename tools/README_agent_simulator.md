# Agent 调用模拟程序

用于模拟外部 Agent 调用后端接口，方便你本地验证：保存后界面是否正常刷新、聊天框是否提示、网格绘制是否正确。

## 使用前

1. 先启动后端：在项目根目录执行 `python backend/server.py`（或按你现有方式启动），保证 `http://127.0.0.1:9000` 可用。
2. 可选：在浏览器打开前端页面，并勾选要查看的网格类型（如任务1、任务3等），便于观察“保存后是否自动更新、位置是否偏移”。

## 运行方式

在项目根目录执行：

```bash
python tools/agent_simulator.py
```

API 地址会从项目根目录的 `config.json` 的 `api_server` 读取（默认 `http://127.0.0.1:9000`）。

## 菜单说明

运行后会出现菜单，只需输入 **0～7** 选接口，无需再输入任何数据（偏移和偏好均已写死在脚本中）：

- **1) 网格保存 - task1**：拉取当前任务1数据，应用固定偏移（lat+0.01, lon+0.02）后 POST `/api/grid/save`（task=task1）。
- **2) 网格保存 - task2**：同上，task2。
- **3) 网格保存 - task3**：同上，task3（含 task3Grid + preferences）。
- **4) 网格保存 - group**：拉取当前分组数据，应用相同偏移后 POST（task=group）。
- **5) 偏好保存**：使用写死的示例偏好（3 组 gridIndex）POST `/api/grid/preference/save`。
- **6) 通道保存**：读取 `grid/channel.json` 并 POST `/api/grid/channel/save`，前端收到 SSE 后显示通道立方体（中心经纬高，长宽 km，高度 m）。
- **7) 任务区域保存**：读取 `grid/task_area.json` 并 POST `/api/grid/task-area/save`，前端收到 SSE 后显示各任务区域矩形（中心经纬度、长宽 km）。
- **0) 退出**。

## 写死的数据（可在脚本顶部修改）

- **网格偏移**：`GRID_OFFSET_LAT = 0.01`，`GRID_OFFSET_LON = 0.02`（单位：度）。选 1/2/3/4 时会在现有经纬度上加上该偏移再保存，地图上网格会移动，便于验证绘制。
- **偏好**：`SAMPLE_PREFERENCES = [[300,301,302], [303,304,305], [306,307,308]]`。选 5 时直接发送该数据。

## 验证绘制是否正常

- 选 **1 / 2 / 3 / 4**：保存后前端轮询会拉取新数据并刷新，地图上网格应**整体移动**（偏移 0.01° 纬、0.02° 经），用于验证绘制与自动更新是否正常。
- 选 **5**：发送示例偏好，用于验证偏好接口与界面偏好显示是否正常。

运行后看终端是否打印“成功”，再在浏览器中确认：聊天框是否出现“数据生成成功，并已保存”及对应接口说明，地图上的网格/偏好是否按预期更新或移动。
