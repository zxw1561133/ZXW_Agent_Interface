DA智能体系统 · 详细通信流程图

谁（函数/模块）请求了谁 → 谁返回了什么

一、页面加载（App.init）
App.init() → fetch → GET {apiBase}/config.json ← 返回 后端读 config.json → JSON { tile_server_public, tiles_dir, api_server } → 前端写入 Config
App.init() → fetch → GET {apiBase}/api/config ← 返回 （config.json 失败时）后端 handle_config() → { tileServer, localTilesPath, apiServer }
App.init() → new → Globe3D('globe3d') / GridSystem(globe3d) / ChatAgent() / PanelManager() / ContextMenu(gridSystem) ← 返回 实例，无返回值；App 持有引用
App.setupEventListeners() → 订阅 → eventBus.on('grid:load', callback) / on('grid:importComplete') / on('globe:zoom') / on('globe:click') ← 返回 无；之后 emit 时 callback 被调用
App.init() → 调用 → this.loadServerMap(true) ← 返回 内部 fetch 瓦片地址 HEAD/GET → 成功则 globe3d.setImageryFromTileServer(base)，失败则 setImageryToNoService()
App.init() → 调用 → apiService.healthCheck() ← 返回 ApiService 内部 fetch GET /api/health → 后端 handle_health() → { status: 'ok', message, version } → App 根据 status 更新状态栏
二、点击「加载网格」（App.loadSelectedGrids）
App.loadSelectedGrids() → 调用 → apiService.getGridDataLikeDA() ← 返回 见下方 ApiService 内部步骤，最终返回 allGridData
ApiService.getGridDataLikeDA() → 内部 4 次 getGridData → fetch GET /api/grid/data?task=task1、task2、task3、group ← 返回 后端 handle_grid_data() 读 Test_grid_task1.json 等 → 4 份 JSON → ApiService 组装为 { initGrid, task1Grid, task2Grid, task3Grid, groups }
App.loadSelectedGrids() → 发布 → eventBus.emit('grid:load', { data: allGridData, taskType }) ← 触发 EventBus 调用已订阅的 callback，传入 { data, taskType }
App 的 grid:load 回调 → 调用 → this.gridSystem.loadGridData(data, taskType) ← 返回 无；GridSystem 内部取 data[taskType] 数组，逐条 createGridEntity(grid, taskType) 画到 Cesium
GridSystem.loadGridData(data, taskType) → 内部循环 → this.createGridEntity(grid, taskType) ← 返回 无；viewer.entities.add(polygon + label)，网格显示在地球上
App.loadSelectedGrids() → 调用 → this.chatAgent.sendSystemMessage('✅ 已加载 N 类网格数据') ← 返回 ChatAgent.addMessage(内容, 'agent') → 聊天框追加一条消息
三、对话里说「是」导入（ChatAgent）
用户 → 输入「是」发送 → ChatAgent.sendMessage() ← addMessage(内容,'user') 显示；因 pendingImport 为 true 走 handleImportResponse(content)
ChatAgent.handleImportResponse('是') → 调用 → this.performImport() ← 返回 无（async）；内部拉网格、发事件、发聊天消息
ChatAgent.performImport() → 调用 → apiService.getGridDataLikeDA() ← 返回 同「二」：4 次 GET /api/grid/data → 后端返回 4 份 JSON → 组装 allGridData 返回
ChatAgent.performImport() → 发布 → eventBus.emit('grid:load', { data: allGridData, taskType }) ← 触发 App 的 grid:load 回调被调用 → gridSystem.loadGridData(data, taskType) → 网格画到地球
ChatAgent.performImport() → 发布 → eventBus.emit('grid:importComplete', { tasks: loaded }) ← 触发 App 的 grid:importComplete 回调 → 把任务选择框选中项设为 tasks
ChatAgent.performImport() → 调用 → this.addMessage('✅ 导入完成！...', 'agent') ← 返回 聊天框追加一条 Agent 消息
四、后端 API（谁处理、返回什么）
浏览器 fetch → GET → /config.json ← 后端 APIHandler.handle_config_file() 读 config.json → 返回 JSON（过滤 _ 开头键）
浏览器 fetch → GET → /api/config ← 后端 handle_config() → { tileServer, localTilesPath, apiServer }
ApiService.healthCheck() → fetch → GET → /api/health ← 后端 handle_health() → { status: 'ok', message: 'Service is running', version: '1.0.0' }
ApiService.getGridData(task) → fetch → GET → /api/grid/data?task=task1（或 task2/task3/group） ← 后端 handle_grid_data(params) 读 grid/Test_grid_task1.json 等 → 返回文件原始 JSON（如 { initGrid, task1Grid }）
