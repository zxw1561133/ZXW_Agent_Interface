# Cesium 瓦片要求说明

本工程 3D 地球使用 **Cesium WebMercator** 瓦片，要求如下。

## 1. 目录结构（必须）

瓦片路径为：**`z/x/y.扩展名`**

- **z**：层级（level），整数，从 0 或 1 开始
- **x**：列号，西→东 增加
- **y**：行号，**北→南** 增加（y=0 为最北一行）

示例：

```
tiles/
├── 1/
│   ├── 0/
│   │   ├── 0.jpg    ← 西北
│   │   └── 1.jpg
│   └── 1/
│       ├── 0.jpg
│       └── 1.jpg    ← 东南
```

## 2. level 1 最少 4 张（必须）

Cesium 在 level 1 会请求这 4 张，缺一不可：

| 路径       | 含义     |
|------------|----------|
| `1/0/0.jpg` | 西北块   |
| `1/0/1.jpg` | 西南块   |
| `1/1/0.jpg` | 东北块   |
| `1/1/1.jpg` | 东南块   |

若只有 `1/0/0.jpg`，其他三块会 404，地球会缺一块或全不显示。

## 3. 扩展名

- 支持 **.jpg** 或 **.png**
- 前端默认请求 `.jpg`，若你只有 `.png`，请在 `frontend/js/core/Config.js` 里把 `TILES.TILE_EXT` 改为 `'png'`

## 4. 行列方向（TMS 与 XYZ）

- **当前默认**：y=0 为北（与 OSM/Google 一致）
- 若你的瓦片是 **TMS**（y=0 在南、从下往上递增），请在 `Config.js` 里把 `TILES.USE_REVERSE_Y` 改为 `true`

## 5. 检测本地数据是否合格

在 `map` 目录下执行：

```bat
python check_tiles.py
```

会检查目录是否存在、z/x/y 结构、level 1 是否齐全、扩展名等，并给出修改建议。

## 6. 在浏览器里自测

先启动瓦片服务（`python server.py`），再在浏览器访问：

- `http://127.0.0.1:9001/1/0/0.jpg`（或你实际用的扩展名）

能打开图片说明路径和扩展名正确；再保证 1/0/1、1/1/0、1/1/1 也能访问。
