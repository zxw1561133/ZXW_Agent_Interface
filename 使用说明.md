# 3D Earth Platform 使用说明

## 快速启动

### 方式一：使用本地瓦片（推荐，无需联网）

**1. 配置地图路径（统一配置）**

在项目根目录编辑 **`config.json`**，所有地图/瓦片路径由此配置，前端、后端、瓦片服务都会读取：

| 字段 | 说明 |
|------|------|
| `tiles_dir` | 瓦片在磁盘上的目录（map 服务用；Windows 双反斜杠 `\\`） |
| `tile_service_url` | 瓦片服务地址，供 9000 代理转发（一般为 `http://127.0.0.1:9001`） |
| `tile_server_public` | 前端使用的瓦片地址（本机示例 `http://127.0.0.1:9001`） |
| `api_server` | 前端 API 地址（本机示例 `http://127.0.0.1:9000`） |

修改后需**重启** backend 与 map 两个服务；前端会通过 `/api/config` 自动拉取最新配置。

**2. 启动地图服务**

```bash
cd map
python server.py
```

**3. 启动后端服务**

```bash
cd backend
python server.py
```

**4. 访问**

浏览器打开 `http://127.0.0.1:9000`

---

### 方式二：使用在线地图（无需本地瓦片，需要联网）

如果不方便使用本地瓦片，可以改用在线卫星地图：

**1. 修改前端配置**

编辑 `frontend/js/core/Config.js`：

```javascript
// 注释掉本地服务
// TILE_SERVER: 'http://127.0.0.1:9001',

// 使用在线地图
TILE_SERVER: 'https://server.arcgisonline.com/ArcGIS/rest/services/World_Imagery/MapServer/tile',
```

**2. 只启动后端**

```bash
cd backend
python server.py
```

**3. 访问**

浏览器打开 `http://127.0.0.1:9000`

## 瓦片数据格式

瓦片目录结构：
```
tiles/
├── 1/              # 缩放级别 1
│   ├── 0/
│   │   ├── 0.jpg
│   │   └── 1.jpg
│   └── 1/
├── 2/              # 缩放级别 2
├── ...
└── 10/             # 缩放级别 10
```

- 格式：`{z}/{x}/{y}.jpg`
- z：缩放级别（0-10）
- x, y：瓦片坐标

## 常见问题

### Q: 地图不显示（蓝色球体）

A: 检查以下几点：
1. 瓦片服务是否启动？`http://127.0.0.1:9001` 能否访问
2. 瓦片路径配置是否正确？查看 `map/server.py` 启动日志
3. 是否有瓦片数据？检查 `tiles/` 目录是否为空

### Q: 如何切换本地/在线地图？

A: 修改 `frontend/js/core/Config.js` 中的 `TILE_SERVER`：
- 本地：`http://127.0.0.1:9001`
- 在线：`https://server.arcgisonline.com/...`

### Q: Windows 双击启动？

A: 双击 `start.bat`（Windows）或 `start.sh`（Linux/Mac）
